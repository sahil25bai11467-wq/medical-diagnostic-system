## Overview

The AI-Driven Expert Medical Diagnostic System accepts two patient inputs — **body temperature (°C)** and **duration of symptoms (days)** — and produces:

1. **Risk Category** (Low / Medium / High) — computed by a trained Decision Tree classifier.
2. **Clinical Diagnosis & Treatment Plan** — generated by a Prolog rule-based expert system.

This project is a BYOP (Bring Your Own Project) capstone submission demonstrating how symbolic AI and machine learning can work together in a single medical application.

---

## Features

- ✅ **Real-time Risk Prediction** using a Decision Tree ML model
- ✅ **Rule-based Diagnosis** via a SWI-Prolog expert knowledge base
- ✅ **Auto Model Training** — trains automatically if no saved model is found
- ✅ **Input Validation** — enforces valid temperature (≥ 35°C) and duration (> 0 days)
- ✅ **Desktop GUI** built with Tkinter (no browser required)
- ✅ **Lightweight & Portable** — minimal dependencies, runs offline

## Installation

### 1. Clone the Repository

```bash
git clone https://github.com/<your-username>/ai-medical-diagnostic-system.git
cd ai-medical-diagnostic-system
```

### 2. Install Python Dependencies

```bash
pip install -r requirements.txt
```

The `requirements.txt` includes:
```
pyswip
scikit-learn
joblib
```

> Tkinter is included in the Python standard library — no extra install needed.

### 3. Verify SWI-Prolog

```bash
swipl --version
# Expected output: SWI-Prolog version 8.x.x ...
```

If this command is not found, revisit the [SWI-Prolog installation guide](https://www.swi-prolog.org/Download.html) for your OS.

## Usage

### Run the Application

```bash
python app.py
```

On first launch, if `risk_classifier.pkl` is not found, the app will **automatically train the ML model** using `symptoms_data.csv`. This takes a few seconds.

### Using the GUI

1. **Enter Body Temperature** — Type the patient's temperature in Celsius (e.g., `38.5`).
2. **Enter Symptom Duration** — Type the number of days symptoms have been present (e.g., `3`).
3. **Click "Diagnose"** — Results appear immediately in the output panel.

## How It Works

### Machine Learning (classifier_model.py)

- A `DecisionTreeClassifier` from **Scikit-learn** is trained on `symptoms_data.csv`.
- Features: `temperature`, `duration_days`
- Target label: `risk_level` (`Low`, `Medium`, `High`)
- The trained model is serialised with **Joblib** as `risk_classifier.pkl`.
- On application start, if the `.pkl` file exists it is loaded directly; otherwise, training runs automatically.

### Symbolic AI / Prolog (rules.pl)

The Prolog knowledge base encodes clinical thresholds:

```prolog
% Fever classification rules
high_fever(Temp) :- Temp >= 39.0.
long_duration(Days) :- Days >= 5.

% Diagnosis rules
diagnose(Temp, Days, 'High Fever - Long Duration', 'Seek immediate medical attention.') :-
    high_fever(Temp), long_duration(Days).

diagnose(Temp, Days, 'High Fever - Short Duration', 'Take antipyretics; rest and hydrate.') :-
    high_fever(Temp), \+ long_duration(Days).

diagnose(Temp, Days, 'Moderate Fever', 'Monitor temperature; rest and fluids recommended.') :-
    \+ high_fever(Temp).
```

Python communicates with SWI-Prolog through the **PySwip** library, submitting queries and receiving bound variable values.

## Training Data

`symptoms_data.csv` contains labelled patient records:

```csv
temperature,duration_days,risk_level
36.5,1,Low
37.8,3,Low
38.2,2,Medium
38.9,5,Medium
39.5,7,High
40.1,3,High
...
```

To retrain the model manually (e.g., after updating the CSV):

```bash
# Simply delete the existing model file and relaunch the app
rm risk_classifier.pkl
python app.py
```
